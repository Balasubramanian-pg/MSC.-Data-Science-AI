## Summary: Proximity Measures for Numerical Attributes

This lecture explains how proximity analysis is performed for **numerical attributes** using distance-based methods. The focus is on:

- Euclidean Distance
    
- Manhattan Distance
    
- Minkowski Distance
    
- Supremum Distance
    

The lecture establishes that for numerical data, similarity is commonly measured through geometric distance calculations.

## Numerical Attributes

Numerical attributes are attributes whose values are numbers.

They can be:

|Type|Meaning|
|---|---|
|Discrete|Finite/countable values|
|Continuous|Infinite/ranged values|

Examples:

|Attribute|Type|
|---|---|
|Age|Numerical|
|Temperature|Numerical|
|Income|Numerical|
|Height|Numerical|

Unlike nominal attributes:

- Arithmetic operations are meaningful
    
- Distances can be computed directly
    

## Core Idea of Numerical Proximity

For numerical attributes:

```text
Closer points → More similar
Farther points → More dissimilar
```

Distance becomes the primary way to measure similarity.

## Euclidean Distance

Euclidean distance is the standard straight-line distance between two points.

The formula:

d(x,y)=\sqrt{\sum_{k=1}^{n}(x_k-y_k)^2}

Where:

|Symbol|Meaning|
|---|---|
|(x_k)|Attribute value of point X|
|(y_k)|Attribute value of point Y|
|(n)|Number of attributes/dimensions|

## Geometric Interpretation

Euclidean distance measures:

```text
Shortest straight-line distance
```

between two points in n-dimensional space.

## Example Dataset

The lecture uses four points:

|Point|A1|A2|
|---|---|---|
|P1|0|2|
|P2|2|0|
|P3|3|1|
|P4|5|1|

## Example: Distance Between P1 and P2

Using Euclidean distance:

d(P1,P2)=\sqrt{(2-0)^2+(0-2)^2}=2.828

Interpretation:

- P1 and P2 are 2.828 units apart.
    

## Important Properties of Euclidean Distance

## 1. Distance to Self = 0

d(x,x)=0

A point is perfectly similar to itself.

## 2. Symmetry

d(x,y)=d(y,x)

Distance remains identical in both directions.

## Need for Normalization

The lecture briefly mentions:

```text
Standardization may be necessary if scales differ.
```

Example:

|Attribute|Scale|
|---|---|
|Age|0-100|
|Income|0-10,000,000|

Without normalization:

- Large-scale features dominate distance calculations.
    

This becomes a major issue in ML pipelines.

## Minkowski Distance

The lecture then introduces:

## Minkowski Distance

This is the generalized form of many distance metrics.

Formula:

d(x,y)=\left(\sum_{k=1}^{n}|x_k-y_k|^r\right)^{1/r}

The parameter:

```text
r
```

controls the type of distance.

## Key Insight

```text
Many common distances are special cases of Minkowski distance.
```

This is one of the most important mathematical ideas in the lecture.

## Case 1: Manhattan Distance

When:

r=1

Minkowski becomes:

## Manhattan Distance

Formula:

d(x,y)=\sum_{k=1}^{n}|x_k-y_k|

Also called:

- Taxicab Distance
    
- L1 Norm
    

## Why "Taxicab" Distance?

The lecture explains:

A taxi cannot move diagonally through buildings.

Instead it moves:

- Horizontally
    
- Vertically
    

Like city blocks.

So Manhattan distance measures actual path traversal distance rather than straight-line distance.

## Example: Manhattan Distance Between P1 and P2

d(P1,P2)=|2-0|+|0-2|=4

Interpretation:

- Travel distance = 4 units
    

instead of Euclidean's 2.828.

## Case 2: Euclidean Distance

When:

r=2

Minkowski becomes:

## Euclidean Distance

This is also called:

- L2 Norm
    

## Case 3: Supremum Distance

When:

r\to\infty

Minkowski becomes:

## Supremum Distance

Formula:

d(x,y)=\max_k |x_k-y_k|

Also called:

- L∞ Norm
    
- Maximum Norm
    

## Interpretation of Supremum Distance

Instead of summing differences:

- Only the largest coordinate difference matters.
    

## Example: Supremum Distance Between P1 and P2

Differences:

```text
|2-0| = 2
|0-2| = 2
```

Maximum difference:

```text
max(2,2)=2
```

So:

d(P1,P2)=2

## Hamming Distance

The lecture briefly notes:

Hamming distance is a special case of Manhattan distance for binary attributes.

Meaning:

- Binary vectors
    
- Compare position-wise mismatches
    

This becomes heavily used in:

- Error correction
    
- NLP
    
- Genomics
    
- Bitwise systems
    

## Comparison of Distance Measures

|Distance Type|Formula Style|Intuition|
|---|---|---|
|Manhattan|Sum of absolute differences|Grid movement|
|Euclidean|Straight-line distance|Geometric shortest path|
|Supremum|Maximum coordinate difference|Worst-case deviation|

## Important Machine Learning Insight

The choice of distance metric changes the behavior of ML algorithms dramatically.

Especially in:

- KNN
    
- Clustering
    
- Recommendation systems
    
- Embedding similarity
    

Different metrics emphasize different geometric properties.

## Hidden Geometric Insight

## Manhattan Distance

Creates:

```text
Diamond-shaped neighborhoods
```

## Euclidean Distance

Creates:

```text
Circular neighborhoods
```

## Supremum Distance

Creates:

```text
Square-shaped neighborhoods
```

This changes clustering boundaries and nearest-neighbour behavior significantly.

## Curse of Dimensionality Connection

The lecture indirectly connects to an important concept:

As dimensionality increases:

- Distance metrics become less informative
    
- Distances begin converging
    
- Neighborhood structure weakens
    

This becomes critical in high-dimensional ML systems.

## Final Takeaways

The lecture establishes a hierarchy of numerical proximity measures.

Core structure:

```text
Minkowski Distance
    ├── Manhattan Distance (r=1)
    ├── Euclidean Distance (r=2)
    └── Supremum Distance (r→∞)
```

Each distance metric captures similarity differently.

The major lesson:

```text
Similarity is geometry-dependent.
```

Choosing the correct distance metric is not a mathematical detail. It fundamentally changes:

- Similarity relationships
    
- Clustering behavior
    
- Classification decisions
    
- ML model performance
    

Euclidean is not universally optimal. The metric must match the structure of the problem domain.
