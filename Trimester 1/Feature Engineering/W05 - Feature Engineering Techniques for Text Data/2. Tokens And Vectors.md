Before we generate features from text, we need to perform a few steps. Text  

cleaning involves several steps. One such step is breaking a sentence down  

into individual words, which is called tokenizing. Another step is to change  

all uppercase to lowercase letters. A third step is to remove what are called  

stop words, which are common words like articles, prepositions, conjunctions,  

etc. They don't add much meaning to the sentences. There's also stemming or  

limitizing, which is for reducing words to their root form, and there are  

libraries available for all of these. So tokenization is the first essential  

step in nearly all NLP pipelines. It means breaking raw text into smaller  

units, typically words, subwords, or sentences. This is quite fundamental  

because machine learning models cannot work with raw text. They need discrete  

units that they can analyse. Once we have tokens, we need to turn them into  

numbers or vectors so algorithms can use them. Vectorization means  

representing tokens in numeric form using techniques like counts or bag of  

words, weighted counts, TFIDF. This is required because machine learning  

models like logistic regression or neural networks need numeric input, and  

without vectorization our models cannot process text at all. So when you  

tokenize text, each token that is a word or a subword is assigned a vector.  

Number of tokens is typically related to the number of vectors. And so for a  

sentence with n tokens, we will typically get n vectors. Plus in some  

cases there are special tokens added if we're using models like bird. When we  

say vectors, what does the vector for a word look like? A vector for a word in  

NLP is simply a list or an array of numbers typically floats that  

represents the semantic properties of that word in a high dimensional space.  

A basic technique to convert words into numeric features is bag of words. This  

counts how often each word appears in a document. It's a basic technique,  

maintains only the count of words. So while it is simple and fast, there is  

no way to preserve anything more complex like context or word order. A  

little smarter technique to convert words into numeric features is TFIDF,  

term frequency, inverse document frequency. At its core, it's a way to  

measure how important a word is to a document relative to a collection of  

documents. TFIDF balances two things term frequency, TF, that's how often a  

word appears in a document. This part rewards words that are frequent in the  

specific document. The second is inverse document frequency or IDF,  

which is how rare the word is across all documents. This part penalises  

words that are common everywhere. By multiplying these two components, TFIDF  

boosts words that are frequent in one document but rare across the corpus.  

For example, in news articles, a word like election will have a higher TFIDF  

score in a political article than in a sports article. This makes TFIDF really  

useful for tasks like text classification or search because it  

helps highlight the terms that truly characterise a document. In practise,  

TFIDF produces a numeric feature vector for each document where each dimension  

corresponds to a word in the vocabulary and the value reflects its importance.  

We will see how to use these techniques to build a text classification model.
