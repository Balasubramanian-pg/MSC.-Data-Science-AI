Corner detection is a fundamental technique in computer vision used to identify "points of interest"—areas in an image where edges intersect. These points are highly informative, as they represent significant intensity variations in multiple directions, unlike edges, which only show variation in one direction.

### 1. Harris Corner Detection

Harris Corner Detection is a classic algorithm that identifies corners by shifting a small window across an image and measuring the change in intensity.

- **The Logic:**
    
    - **Corner:** Significant intensity change in **both X and Y** directions.
        
    - **Edge:** Significant change in **only one** direction.
        
    - **Flat Region:** Little to no change in any direction.
        
- **Mathematical Foundation:** It utilizes the **autocorrelation matrix** (structure tensor) to assign a "corner response" score to each pixel.
    
- **Key Steps:**
    
    1. Compute image gradients ($X$ and $Y$ directions).
        
    2. Build the structure tensor.
        
    3. Calculate the corner response score for each pixel.
        
    4. Threshold the response to isolate strong candidate points.
        
    5. Apply Non-Maximum Suppression (NMS) to ensure only the best-defined corners remain.
        

### 2. Shi-Tomasi Corner Detection

Shi-Tomasi is an improvement on Harris, designed to be more stable and repeatable in practical scenarios.

- **The Logic:** Instead of the Harris response formula, Shi-Tomasi uses the **minimum eigenvalue** of the local structure tensor.
    
- **Why it's better:** Mathematically, a corner is confirmed if both eigenvalues of the structure tensor are large. By simply taking the smaller of the two eigenvalues ($\min(\lambda_1, \lambda_2)$), the algorithm provides a highly robust metric that is often less sensitive to parameter tuning than the classic Harris response.
    

### 3. Practical Implementation Comparison

|**Feature**|**Harris Corner Detection**|**Shi-Tomasi Corner Detection**|
|---|---|---|
|**Logic**|Harris response formula|Minimum eigenvalue ($\min(\lambda_1, \lambda_2)$)|
|**Sensitivity**|Highly dependent on $k$ and block size|More stable and repeatable|
|**Common API**|`cv2.cornerHarris()`|`cv2.goodFeaturesToTrack()`|
|**Performance**|Classic, effective|Generally preferred in modern pipelines|

### 4. Implementation Example (OpenCV)

**Harris Corner Detection Snippet:**

Python

```
import cv2
import numpy as np

## Convert to float32 as required by OpenCV
gray_float = np.float32(gray_image)

## Harris Corner Detection
dst = cv2.cornerHarris(gray_float, blockSize=2, ksize=3, k=0.04)

## Dilate to mark corners clearly
dst = cv2.dilate(dst, None)

## Threshold to isolate strong corners
gray_image[dst > 0.01 * dst.max()] = [0, 0, 255]
```

**Shi-Tomasi Corner Detection Snippet:**

Python

```
## 'goodFeaturesToTrack' implements Shi-Tomasi
corners = cv2.goodFeaturesToTrack(gray_image, maxCorners=40, qualityLevel=0.01, minDistance=2)

## Convert to integer coordinates for plotting
corners = np.int0(corners)
```

- **When to use Harris:** Use it when you need a classic, well-understood mathematical approach for research or fundamental image processing tasks.
    
- **When to use Shi-Tomasi:** Use it for reliable, real-world object tracking and image stitching applications. The `goodFeaturesToTrack` function is generally the preferred choice due to its stability and parameter control (like `minDistance` to avoid detecting clustered points).
